Docker Scout
-------------
docker scout cve <image-name> --only-severity critical,high
docker scout disable
docker scout enable

docker scout policy <image-name>

# Scan local image (without pulling)

docker scout cve <image-name> --local
docker scout recommendations <image-name>

## Example Workflow

# Build your image
docker build -t my-app:v1 .

# Scan for vulnerabilities
docker scout cve my-app:v1

# Check if it meets your security policy
docker scout policy my-app:v1
docker scout quickview 


# Compare with previous version
docker scout compare my-app:v1 my-app:v2

# Login to Docker Hub via Command Line
docker login
docker login -u <your-username> -p <your-password>

Using personal access token (recommended)
Go to Docker Hub → Account Settings → Security

Click "New Access Token"

Generate token with appropriate permissions

Use token as password:
docker login -u <your-username>
# When prompted for password, use the access token

Verify login
docker info | grep Username

## GitHub Actions example
 - name: Login to Docker Hub
  uses: docker/login-action@v2
  with:
    username: ${{ secrets.DOCKERHUB_USERNAME }}
    password: ${{ secrets.DOCKERHUB_TOKEN }}

- name: Build and push
  run: |
    docker build -t username/image:tag .
    docker push username/image:tag
